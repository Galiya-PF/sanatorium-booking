# Система автоматизации заявок на санаторно-курортное лечение

Веб-приложение для автоматизации процесса подачи заявок работников на санаторно-курортное лечение и их обработки администратором.

## Функционал

### Для работников:
- Подача заявки на санаторно-курортное лечение через веб-форму
- Выбор санатория из списка
- Указание даты начала и количества дней (7-10 дней)
- Обязательное подтверждение соответствия графику отпусков
- Получение уведомлений о статусе заявки

### Для администратора:
- Просмотр всех заявок в единой таблице
- Статистика по заявкам (ожидающие/одобренные/отклоненные)
- Обработка заявок (одобрение/отклонение)
- Добавление комментариев к заявкам
- Автоматическая генерация уведомлений для работников

## Технологии

- **Backend:** Python + Flask ИЛИ Node.js + Express (оба варианта доступны)
- **Database:** SQLite
- **Frontend:** HTML + JavaScript + Bootstrap 5
- **API:** REST API

## Установка и запуск

### Вариант 1: Python + Flask (рекомендуется)

#### Требования:
- Python 3.7+

#### Установка:

1. Установите зависимости:
```bash
pip install flask
```

2. Запустите сервер:
```bash
python app.py
```

Сервер запустится на `http://localhost:5000`

### Вариант 2: Node.js + Express

#### Требования:
- Node.js 14+

#### Установка:

1. Установите зависимости:
```bash
npm install
```

2. Запустите сервер:
```bash
npm start
```

Сервер запустится на `http://localhost:3000`

### Использование

1. Откройте браузер и перейдите по адресу `http://localhost:5000` (Python) или `http://localhost:3000` (Node.js)
2. На вкладке "Подать заявку" работники могут заполнить форму
3. На вкладке "Админ-панель" администратор может просматривать и обрабатывать заявки

## API Endpoints

### GET /api/sanatoriums
Получить список доступных санаториев

### POST /api/applications
Создать новую заявку

Тело запроса:
```json
{
  "full_name": "Иванов Иван Иванович",
  "sanatorium": "Арман",
  "start_date": "2027-01-01",
  "days": 10,
  "email": "email@example.com",
  "phone": "+7 (XXX) XXX-XX-XX",
  "department": "Бухгалтерия",
  "vacation_schedule_confirmed": true
}
```

### GET /api/applications
Получить список всех заявок (для админа)

### GET /api/applications/:id
Получить информацию о конкретной заявке

### PUT /api/applications/:id
Обновить статус заявки

Тело запроса:
```json
{
  "status": "approved",
  "admin_comment": "Комментарий администратора"
}
```

### DELETE /api/applications/:id
Удалить заявку

### GET /api/stats
Получить статистику по заявкам

## Валидация

- Количество дней: от 7 до 10
- Обязательно подтверждение соответствия графику отпусков
- Все обязательные поля должны быть заполнены

## Пример уведомления работника

При одобрении заявки работник получает уведомление:

> Уважаемый Айгуль! вам забронирована санаторно-курортное лечение в санаторий "Арман" с 01.01.2027 года на 10 дней. Просим подойти в профсоюз за доверенностью

## База данных

### Таблица applications
- id - уникальный идентификатор
- full_name - ФИО работника
- sanatorium - название санатория
- start_date - дата начала лечения
- days - количество дней
- email - email работника
- phone - телефон работника
- department - отдел/подразделение
- vacation_schedule_confirmed - подтверждение графика отпусков
- status - статус (pending/approved/rejected)
- admin_comment - комментарий администратора
- created_at - дата создания заявки
- updated_at - дата обновления заявки

### Таблица sanatoriums
- id - уникальный идентификатор
- name - название санатория
- address - адрес
- contact_phone - контактный телефон
- is_active - активность санатория

## Дополнительная информация

База данных SQLite создается автоматически при первом запуске сервера. Все данные сохраняются локально в файле `sanatorium.db`.

Для настройки email-уведомлений (отправка реальных писем на email) необходимо настроить модуль nodemailer в файле `server.js`.
